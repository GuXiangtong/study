# 需求

## 1. 题目录入

- 用户可以通过**上传图片**（拍照或扫描的试卷/错题）或**手工输入文字**两种方式录入题目。
- 上传图片后，系统应能识别并提取题目内容。

## 2. 题目管理

- 用户可以对已录入的题目进行**修改**和**删除**操作。
- 支持按学科、考试、日期等维度浏览和检索题目。

## 3. 子问题支持

- 每道题目可以包含**若干子问题**（如一道大题下的第(1)(2)(3)小问）。
- 每个子问题独立记录、独立分析。

## 4. 错题信息记录

- 对于每道题目或子问题，用户可以录入：
  - **错误答案**：当时作答的错误内容。
  - **出错原因**：学生自己总结的为何做错的原因。

## 5. 错因分析

- 每道题目提供一个**「分析」按钮**。
- 点击后按照 [CLAUDE.md](./CLAUDE.md) 定义的四步流程进行分析：
  1. 分析错题，定位知识点
  2. 诊断薄弱环节，梳理知识体系
  3. 引导思考，给出解题路径
  4. 生成巩固练习（3 道）
- 分析结果保存至 `错题分析/` 目录，按学科和日期组织。

## 6. 巩固练习与正确答案

- 每道题目提供一个**「生成练习」按钮**，一键生成 3 道同类变式题（基础题、提高题、难题各一）。
- 每道题目可以**查看正确答案**及详细解题步骤。
